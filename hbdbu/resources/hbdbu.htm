HbDBU - Harbour's Database Utility

HbDBU is an application which can be used to mange tables created with DBFCDX, DBFNTX and DBFNSX database drivers as default and any other drivers linked to the applications.

HbDBU is based on COM like component HbQtDBU. HbQtDBU exposes the methods and event handlers to configure HbDBU to any level. HbQtDBU can be embedded in any application. It just needs a parent widget to expose itself. It has a expanded set of methods which application can exploit to manipulate its interface and execution behavior.

HbDBU code contains code to use ADS ( Advantage Database Server ) and CACHERDD ( http://cacherdd.vouch.info ) drivers in addition to default drivers. The relevant code is guarded with #define constants so that a plain default HbDBU can be compiled and linked. The other drivers code is retained here is to let the developers could grasp the idea and tune up HbDBU to configure with their own drivers.

HbDBU can be made the default program to execute .dbu and .dbf files in the Windows explorer. QtContrib intsllation setup does not do it automatically.

 

How to Invoke

C:/>HbDBU  [.dbu]  [ [cDriver] .dbf1 [.dbf2 [.dbfN ] ] ]  [ [cDriver2] .dbf1 [.dbf2 [.dbfN ] ] ]

If one or more .dbf(s) is/are supplied on the command line and .dbu is not supplied, then no environment file will be read and written.

If no <cDriver> is supplied on the command line then last assigned driver on the interface will be used. If no .dbu is restrored, then DBFCDX driver becomes the default driver.

If more then one .dbu(s) are supplied on the command line then the last encountered .dbu will be restored.

If no .dbu is supplied and no .dbf is supplied on the command line, then "settings.dbu" at OS specific user-data location will be made default environment file.
